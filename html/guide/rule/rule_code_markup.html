Publishing Conventions

코드규칙

코드 들여쓰기

  • 1탭 들여쓰기 하며, 1탭의 크기는 공백 4칸으로 설정한다.

들여쓰기를 하지 않는 경우

주석 / 빈줄

  • 가독성을 고려하여 빈 줄을 추가한다. 단, 1줄을 초과하지 않는다.
  • 주석은 시작과 종료를 표시한다.
  • 코멘트가 있을경우 Start 주석 아래에 "TODO" 한줄 추가하여 주석처리

수정/추가/삭제

  • 조치사항 진행 결과의 주석을 남긴다. (예 : [update or 수정][날짜][조치내용][이슈])
  • 잦은 수정사항을 고려하여 다중구조인 경우만 닫기주석으로 범위를 표시한다.

한줄 수정

영역 수정

클래스나 단순 영역 수정

  • 수정 종료태그에는 변경 상세내용은 제외

단일항목 이슈단위 표시

  • 조치사항 이슈가 많은경우 이슈단위로 주석을 남긴다. (예 : [제목][조치내용][이슈])

Attribute 지정 순서

기능 > 이름 > 정보 > 선택속성 > 클래스 > 상태 순서로 적용한다.
