네이밍 규칙

일반규칙

  • 이름은 영문 소문자, 숫자, 언더스코어(_), 하이픈(-)으로 작성한다.
  • 시작이름은 영문 대문자, 숫자로 시작할 수 없다.

표기법

  • 일반 클래스명과 스크립트에서 사용되는 클래스명의 동일사용을 지양한다.
  • 클래스 예약어는 상황에 맞게 선택적으로 참고한다.
  • js-접두어를 사용하는 클래스는 디자인을 위한 CSS로 해당 클래스에 스타일을 정의하지 않는다.
  • 단, 공통적으로 기능을 정의하는 경우에는 별도의 CSS로 분리하도록 한다.

파일 및 폴더 네이밍 규칙

  • 스네이크 표기법(underbar_nate)을 사용한다.
  • 파스칼 표기법은 사용하지 않는다.
  • * 파스칼 표기법 : 모든 단어에서 첫번째 문자는 대문자이며 나머지는 소문자이다.(예: PascalCase)
  • * 카멜 표기법 : 각 단어의 첫문자를 대문자로 표기하고 붙여쓰되, 맨처음 문자는 소문자로 표기한다.(예: PascalCase)

Layout Selector(ID/Class)

  • 개발 케이스에 따라서 중복 아이디가 사용될 수 있으므로 최상위 레이아웃, 폼 요소, 책갈피를 제외하고 id를 사용하지 않는다.
  • 레이어 팝업의 경우 각 팝업의 이름을 제외하고 절대 공통아이디로 사용 할 수 없다.
  • 최상위 레이아웃, 폼 요소, 책갈피를 제외하고 ID를 사용하지 않는다.

단계 규칙

  • lnb, anb, mnb, snb의 메뉴구조는 클래스의 통일성을 갖춘다. (메뉴구조 복붙 고려)
  • fnb, gnb, unb, qnb의 메뉴구조는 클래스를 생략 할 수 있다.

Class Selector

  • 체인 선택자(멀티클래스)의 개수는 3개까지 권장한다.
  • 클래스 예약어는 상황에 맞게 선택적으로 참고한다.
  • 레이아웃에 한하여 정의된 예시이며, 자세한 내용은 설계가이드를 참고한다.
